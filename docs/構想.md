# 構想

GitHubのリポジトリを管理する。一つのIssueには一つのリポジトリが紐づきます。それをGitHub Projectsのダッシュボードでステータスやラベルで現状を可視化したいです。
下記の構想を参考に作成してください。

---

## 🎯 目標の最終形

あなたのやりたい構成は次の通り：

| 要素       | 内容                                                     |
| -------- | ------------------------------------------------------ |
| **管理対象** | KenichiroAraiユーザーの全リポジトリ（自動取得、最大100件）                  |
| **管理場所** | 専用の「管理リポジトリ」（repo-dashboard）                           |
| **表現方法** | 各リポジトリ = 1つのIssue（タイトル＝リポジトリ名）                         |
| **可視化**  | GitHub Projects（ボードやテーブル）でステータスを表示                     |
| **更新方法** | GitHub Actionsで毎日自動的に各リポジトリの情報を取得し、Issueを自動更新（手動実行も可能） |
| **目的**   | どのリポジトリが活発か／放置されているかを一目で把握する                           |

GitHub Projectでは下のようなダッシュボードを構築：

| リポジトリ  | Stars | Open Issues | 最終Push       | ステータス       |
| ------ | ----- | ----------- | ------------ | ------------ |
| repo-A | ⭐ 24  | 🐞 3        | ⏰ 2025-11-05 | 🟢 アクティブ    |
| repo-B | ⭐ 5   | 🐞 0        | ⏰ 2025-09-21 | 🔴 停滞中      |

### ② Secretの登録

**不要です。** ワークフローでは自動的に提供される `GITHUB_TOKEN` を使用します。

※ ただし、100件以上のリポジトリを管理する場合や、より高度な権限が必要な場合は、**Settings → Secrets → Actions** に Personal Access Token を追加することも可能です。

---

### ③ GitHub Actions ワークフロー

`.github/workflows/repo-status-sync.yml` に以下を保存：

```yaml
# リポジトリ状態同期ワークフロー
# KenichiroAraiの全リポジトリの状態を定期的に取得してIssueとして管理する
name: Sync Repository Status to Issues

on:
  schedule:
    - cron: "0 0 * * *" # 毎日午前9時（JST）に実行（UTC 0:00）
  workflow_dispatch: # 手動実行も可能

jobs:
  sync-repo-status:
    runs-on: ubuntu-latest
    permissions:
      issues: write
      contents: read
    steps:
      # ステップ0: リポジトリをチェックアウト（gh CLIを使用するため）
      - name: リポジトリをチェックアウト
        uses: actions/checkout@v4

      # ステップ1: KenichiroAraiユーザーの全リポジトリを取得
      - name: KenichiroAraiの全リポジトリを取得
        id: repos
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          echo "KenichiroAraiの全リポジトリを取得中..."

          # gh apiコマンドを使用してリポジトリ一覧を取得（より信頼性が高い）
          # per_page=100で最大100件まで取得
          response=$(gh api /users/KenichiroArai/repos?per_page=100 --paginate)

          # デバッグ: APIレスポンスの最初の部分を表示
          echo "APIレスポンス（最初の500文字）:"
          echo "$response" | head -c 500
          echo ""

          # リポジトリのfull_nameを抽出
          repos=$(echo "$response" | jq -r '.[].full_name' | tr '\n' ' ')

          # 環境変数に保存して次のステップで使用できるようにする
          echo "repos=$repos" >> $GITHUB_ENV
          echo "取得したリポジトリ: $repos"

      # ステップ2: 各リポジトリの状態をIssueに同期
      - name: リポジトリ状態をIssueに同期
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          # リポジトリが取得できているか確認
          if [ -z "$repos" ]; then
            echo "エラー: リポジトリリストが空です"
            exit 1
          fi

          # 取得した全リポジトリをループ処理
          for repo in $repos; do
            echo "リポジトリ $repo を処理中..."

            # GitHub APIからリポジトリの詳細情報を取得（gh apiを使用）
            data=$(gh api /repos/$repo)

            # 必要な情報をJSONから抽出
            name=$(echo "$data" | jq -r .name)
            stars=$(echo "$data" | jq -r .stargazers_count)
            open_issues=$(echo "$data" | jq -r .open_issues_count)
            pushed_at=$(echo "$data" | jq -r .pushed_at)

            # 最終Push日時を日付形式に変換
            pushed_date=$(date -d "$pushed_at" +%Y-%m-%d)

            # 最終Push日からの経過日数を計算
            now=$(date -u +%s)
            pushed_sec=$(date -d "$pushed_at" +%s)
            diff_days=$(( (now - pushed_sec) / 86400 ))

            # 経過日数によってステータスを決定
            if [ $diff_days -le 7 ]; then
              # 7日以内: アクティブ
              status="🟢 アクティブ"
            elif [ $diff_days -le 30 ]; then
              # 8-30日: 最近アクティブ
              status="🟡 最近アクティブ"
            else
              # 30日以上: 停滞中
              status="🔴 停滞中"
            fi

            # Issueの本文を作成
            body="### 📦 リポジトリ: [$repo](https://github.com/$repo)
            - ⭐ スター数: **$stars**
            - 🐞 未解決Issue数: **$open_issues**
            - ⏰ 最終Push日: **$pushed_date**
            - 🏷️ ステータス: **$status**"

            # 既存のIssueを検索（タイトルで検索）
            issue_number=$(gh issue list --repo "$GITHUB_REPOSITORY" --search "$name in:title" --json number --jq '.[0].number' 2>/dev/null || echo "")

            # Issue番号が取得できなければ新規作成、あれば更新
            if [ -z "$issue_number" ]; then
              echo "$repo の新しいIssueを作成中..."
              gh issue create --repo "$GITHUB_REPOSITORY" \
                --title "$name" \
                --body "$body"
            else
              echo "$repo の既存Issue (#$issue_number) を更新中..."
              gh issue edit "$issue_number" --repo "$GITHUB_REPOSITORY" \
                --body "$body"
            fi
          done
```

---

### ④ 初回手動実行

GitHub → Actions → 「Sync Repository Status to Issues」 → "Run workflow"

これで、各リポジトリごとにIssueが自動生成されます👇

---

### ⑤ Projectで可視化

1. 「Projects」 → 「New project」 → **Table view**
2. 自動的に作成されたIssueを追加
3. 必要に応じて「Status」「Assignees」などのカスタムフィールドを追加

すると、

* 各リポジトリが1カード（＝1Issue）
* ステータスは各Issueの本文に表示（🟢 アクティブ / 🟡 最近アクティブ / 🔴 停滞中）
* Issueタイトルはリポジトリ名
  としてダッシュボード化できます。
