# 構想

GitHubのリポジトリを管理する。一つのIssueには一つのリポジトリが紐づきます。それをGitHub Projectsのダッシュボードでステータスやラベルで現状を可視化したいです。
下記の構想を参考に作成してください。

---

## 🎯 目標の最終形

あなたのやりたい構成は次の通り：

| 要素       | 内容                                                |
| -------- | ------------------------------------------------- |
| **管理対象** | 複数のリポジトリ（例：A, B, C...）                            |
| **管理場所** | 専用の「管理リポジトリ」                                      |
| **表現方法** | 各リポジトリ = 1つのIssue                                 |
| **可視化**  | GitHub Projects（ボードやテーブル）でラベル・ステータスを表示            |
| **更新方法** | GitHub Actionsで定期的（例：毎日）に各リポジトリの情報を取得し、Issueを自動更新 |
| **目的**   | どのリポジトリが活発か／放置されているかを一目で把握する                      |

---

## 🧩 構成イメージ

```text
+---------------------------------------+
| repo-dashboard （管理リポジトリ）     |
| ├── .github/workflows/update.yml      |
| ├── Project（Dashboard）              |
| └── Issues                            |
|       ├── repo-A 状況                 |
|       ├── repo-B 状況                 |
|       └── repo-C 状況                 |
+---------------------------------------+
```

GitHub Projectでは下のようなダッシュボードを構築：

| リポジトリ  | Stars | Open Issues | 最終Push       | ステータス     |
| ------ | ----- | ----------- | ------------ | --------- |
| repo-A | ⭐ 24  | 🐞 3        | ⏰ 2025-11-05 | 🟢 Active |
| repo-B | ⭐ 5   | 🐞 0        | ⏰ 2025-09-21 | 🔴 Stale  |

## ⚙️ 実装ステップ詳細

### ① 専用リポジトリ作成

例）
`kenichiro-arai/repo-dashboard`

---

### ② Secretの登録

**Settings → Secrets → Actions** に以下を追加：

* `GH_TOKEN` : Personal Access Token
  （スコープ：`repo`, `read:org`, `project`）

---

### ③ GitHub Actions ワークフロー

`.github/workflows/update.yml` に以下を保存：

```yaml
name: Update Repository Issues Dashboard

on:
  schedule:
    - cron: "0 0 * * *"  # 毎日午前9時（JST）に実行（UTC 0:00）
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - name: Set repositories
        id: repos
        run: |
          echo "repos=user/repo-A user/repo-B user/repo-C" >> $GITHUB_ENV

      - name: Update or create issues
        env:
          GH_TOKEN: ${{ secrets.GH_TOKEN }}
        run: |
          for repo in $repos; do
            echo "Processing $repo ..."
            data=$(curl -s -H "Authorization: token $GH_TOKEN" \
              https://api.github.com/repos/$repo)

            name=$(echo "$data" | jq -r .name)
            stars=$(echo "$data" | jq -r .stargazers_count)
            open_issues=$(echo "$data" | jq -r .open_issues_count)
            pushed_at=$(echo "$data" | jq -r .pushed_at)
            pushed_date=$(date -d "$pushed_at" +%Y-%m-%d)
            now=$(date -u +%s)
            pushed_sec=$(date -d "$pushed_at" +%s)
            diff_days=$(( (now - pushed_sec) / 86400 ))

            if [ $diff_days -le 7 ]; then
              status="🟢 Active"
              label="active"
            elif [ $diff_days -le 30 ]; then
              status="🟡 Recently Active"
              label="recent"
            else
              status="🔴 Stale"
              label="stale"
            fi

            body="### 📦 Repository: [$repo](https://github.com/$repo)
- ⭐ Stars: **$stars**
- 🐞 Open Issues: **$open_issues**
- ⏰ Last Push: **$pushed_date**
- 🏷️ Status: **$status**
"

            gh issue list --label "$repo" --json number --jq '.[0].number' > issue.txt || true
            issue_number=$(cat issue.txt)

            if [ -z "$issue_number" ]; then
              echo "Creating new issue for $repo"
              gh issue create --title "📊 $repo Status" \
                --body "$body" \
                --label "$repo,$label,auto-report"
            else
              echo "Updating existing issue for $repo (#$issue_number)"
              gh issue edit "$issue_number" --body "$body" --add-label "$label" --remove-label "active,recent,stale"
            fi
          done
```

---

### ④ 初回手動実行

GitHub → Actions → 「Update Repository Issues Dashboard」 → “Run workflow”

これで、各リポジトリごとにIssueが自動生成されます👇

---

### ⑤ Projectで可視化

1. 「Projects」 → 「New project」 → **Table view**
2. 「Repository」「Stars」「Status」などの列を追加
3. フィルタリング条件で `label:auto-report` を選択

すると、

* 各リポジトリが1カード（＝1Issue）
* ステータスはラベル（active / recent / stale）
  としてダッシュボード化できます。
