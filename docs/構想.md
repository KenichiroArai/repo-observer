# 構想

GitHubのリポジトリを管理する。一つのIssueには一つのリポジトリが紐づきます。それをGitHub Projectsのダッシュボードでステータスやラベルで現状を可視化したいです。
下記の構想を参考に作成してください。

---

## 🎯 目標の最終形

あなたのやりたい構成は次の通り：

| 要素       | 内容                                                     |
| -------- | ------------------------------------------------------ |
| **管理対象** | KenichiroAraiユーザーの全リポジトリ（自動取得、最大100件）                  |
| **管理場所** | 専用の「管理リポジトリ」（repo-dashboard）                           |
| **表現方法** | 各リポジトリ = 1つのIssue（タイトル＝リポジトリ名）                         |
| **可視化**  | GitHub Projects（ボードやテーブル）でステータスを表示                     |
| **更新方法** | GitHub Actionsで毎日自動的に各リポジトリの情報を取得し、Issueを自動更新（手動実行も可能） |
| **目的**   | どのリポジトリが活発か／放置されているかを一目で把握する                           |

## ⚙️ 実装ステップ詳細

### ① 専用リポジトリ作成

例）
`kenichiro-arai/repo-dashboard`

---

### ② Secretの登録

**基本的には不要です。** ワークフローでは自動的に提供される `GITHUB_TOKEN` を使用します。

**ただし、GitHub Project連動を利用する場合：**

Project操作には追加の権限が必要な場合があります。その場合は以下の設定を推奨します：

1. Personal Access Token (Classic) を作成
   - **Settings** → **Developer settings** → **Personal access tokens** → **Tokens (classic)**
   - 必要な権限：`repo`, `project` (read:projectも含む)
2. リポジトリのSecretsに登録
   - リポジトリの **Settings** → **Secrets and variables** → **Actions**
   - **New repository secret** をクリック
   - Name: `PROJECT_TOKEN`
   - Secret: 作成したPersonal Access Token

※ 100件以上のリポジトリを管理する場合も、Personal Access Token の使用を検討してください。

---

### ③ GitHub Actions ワークフロー

`.github/workflows/repo-status-sync.yml` に以下を保存：

**主な機能：**
- 毎日自動実行（UTC 18:00 = JST 3:00）または手動実行
- 対象ユーザー、Project番号、ステータスフィールド名を手動実行時に指定可能
- リポジトリの詳細情報を取得（説明、言語、ライセンス、トピックス、リリース情報など）
- 5段階のステータス判定（頻繁に更新、定期的に更新、時々更新、更新が少ない、停滞中）
- Push日とIssue更新日の両方を考慮した活動判定
- GitHub Projectとの自動連動（GraphQL API使用）
- closedなIssueはProjectステータス更新をスキップ

詳細な実装は[repo-status-sync.yml](../.github/workflows/repo-status-sync.yml)を参照してください。

---

### ④ 初回手動実行

GitHub → Actions → 「リポジトリ状態をIssueに同期」 → "Run workflow"

**手動実行時のパラメータ：**
- **対象ユーザー名**：取得するリポジトリのユーザー名（デフォルト: KenichiroArai）
- **Project番号**：連動するGitHub ProjectのID（デフォルト: 15、未指定でProject連動なし）
- **Projectのステータスフィールド名**：Projectで使用するステータスフィールドの名前（デフォルト: Status）

これで、各リポジトリごとにIssueが自動生成され、指定したProjectに追加されます👇

---

### ⑤ Projectで可視化

1. 「Projects」 → 「New project」 → **Table view**
2. ワークフローで自動的にIssueがProjectに追加されます（PROJECT_TOKENが設定されている場合）
3. ステータスフィールド「Status」を追加し、以下のオプションを設定：
   - 頻繁に更新（7日以内）
   - 定期的に更新（8-30日）
   - 時々更新（31-180日）
   - 更新が少ない（181-365日）
   - 停滞中（366日以上）

すると、

* 各リポジトリが1カード（＝1Issue）
* ステータスは自動的に更新される（最終Push日またはIssue更新日で判定）
* Issueタイトルはリポジトリ名
* Issueの本文には詳細情報が表形式で整理されて表示
  としてダッシュボード化できます。
